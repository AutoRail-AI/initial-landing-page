# AutoRail Brand Guidelines

This document provides comprehensive brand guidelines for AutoRail, including logos, colors, typography, and usage rules. Use this as the definitive reference for all brand-related decisions.

**Mission Statement:** Infrastructure on Autopilot for AI Agents (Agentic Systems)

**Brand Positioning:** AutoRail bridges the gap between rapid frontend generation and production-grade infrastructure. We automatically provision the backend primitives that real products need—stateful memory, workflow orchestration, guardrails, and failure-safe execution patterns.

## Typography

### Primary Font: Space Grotesk

**Usage:** The main typeface for headlines, UI elements, and brand messaging. A modern, technical sans-serif that conveys precision and innovation.

**Specifications:**
- Font Family: `Space Grotesk`
- Weights Available: 300 (Light), 400 (Regular), 500 (Medium), 600 (Semi Bold), 700 (Bold)
- Style: Geometric sans-serif, technical, modern
- License: Open Font License (Google Fonts)

**Applications:**
- H1, H2, H3 headings
- Navigation menus
- Button labels
- Logo wordmark
- Marketing headlines
- Dashboard titles
- UI labels

**Weight Usage:**
- **700 (Bold):** H1 headlines, primary CTAs
- **600 (Semi Bold):** H2 section headers, emphasis
- **500 (Medium):** H3 subsection headers, labels
- **400 (Regular):** Body text, descriptions

**Web Implementation:**
```css
@import url('https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@300;400;500;600;700&display=swap');

h1, h2, h3 {
  font-family: 'Space Grotesk', sans-serif;
}

h1 { font-weight: 700; font-size: 3.5rem; }
h2 { font-weight: 600; font-size: 2.5rem; }
h3 { font-weight: 500; font-size: 1.75rem; }
```

**Type Scale:**
- H1: 56px / 3.5rem (Bold)
- H2: 40px / 2.5rem (Semi Bold)
- H3: 28px / 1.75rem (Medium)
- Large: 24px / 1.5rem (Medium)
- Body: 16px / 1rem (Regular)
- Small: 14px / 0.875rem (Regular)
- Caption: 12px / 0.75rem (Regular)

---

## Typography Guidelines

### Hierarchy

```
H1 (Space Grotesk Bold, 56px) → Main page title, hero headlines
H2 (Space Grotesk Semi Bold, 40px) → Section headers
H3 (Space Grotesk Medium, 28px) → Subsection headers
H4 (Space Grotesk Medium, 20px) → Component titles
Body (Inter Regular, 16px) → Content text, descriptions
Code (JetBrains Mono Regular, 14px) → Technical content
Caption (Inter Regular, 12px) → Labels, metadata
```

### Line Height
- **Headlines (Space Grotesk):** 1.2
- **Body Text (Inter):** 1.6
- **Code (JetBrains Mono):** 1.5
- **Captions:** 1.4

### Letter Spacing
- **Large Headlines (>40px):** -0.02em (tighter)
- **Headlines (28-40px):** -0.01em
- **Body Text (Inter):** -0.011em
- **Small Text (<14px):** 0em (default)
- **Code:** 0em (default monospace)

### Color Pairings

**Dark Backgrounds:**
- Primary text: Cloud White (`#FAFAFA`)
- Secondary text: Medium grey (`#9CA3AF`)
- Accent text: Electric Cyan (`#00E5FF`)
- Links: Quantum Violet (`#8134CE`)

**Light Backgrounds:**
- Primary text: Void Black (`#0A0A0F`)
- Secondary text: Dark grey (`#4B5563`)
- Accent text: Deep Rail Purple (`#6E18B3`)
- Links: Deep Rail Purple (`#6E18B3`)

### Do's & Don'ts

✅ **Do:**
- Use Space Grotesk for all headlines and UI elements
- Use Inter for body text and documentation
- Use JetBrains Mono for all code examples
- Maintain consistent hierarchy throughout
- Ensure proper line height for readability (1.6 for body text)
- Use appropriate font weights for emphasis
- Test readability at different screen sizes

❌ **Don't:**
- Mix more than three font families
- Use Space Grotesk for long-form body text
- Use decorative or script fonts
- Set body text below 14px
- Use all caps for long text
- Stretch or condense fonts artificially
- Forget to optimize web font loading
- Use too many font weights in one layout

## Usage Guidelines

### Logo Usage Rules

#### Do's ✅
- **Maintain Aspect Ratio:** Never stretch or distort the logo
- **Use Correct Color Versions:** Ensure sufficient contrast with background
- **Provide Clear Space:** Minimum 30% of logo height on all sides
- **Use Vector Files:** Always use SVG when possible for scalability
- **Test at Minimum Sizes:** Ensure legibility at 120px width minimum
- **Align Properly:** Use consistent alignment (left, center, or right)

#### Don'ts ❌
- **Don't Alter Colors:** Never change the gradient or brand colors
- **Don't Add Effects:** No drop shadows, glows, bevels, or filters
- **Don't Rotate:** Keep logos horizontal (except intentional design)
- **Don't Place on Busy Backgrounds:** Ensure clean, uncluttered background
- **Don't Compress Excessively:** Maintain quality in all formats
- **Don't Combine with Other Logos:** Maintain clear separation from partners
- **Don't Recreate:** Always use official logo files

---

### Color Usage Rules

#### Do's ✅
- **Follow the Palette:** Use only defined brand colors
- **Ensure Contrast:** Meet WCAG AA accessibility standards (4.5:1 minimum)
- **Use Gradients Consistently:** Apply gradients in defined directions
- **Test on Devices:** Verify colors render correctly across screens
- **Consider Context:** Dark mode vs. light mode color choices
- **Use System Colors:** Apply success/warning/error colors appropriately

#### Don'ts ❌
- **Don't Invent Colors:** Stay strictly within the defined palette
- **Don't Use Low Contrast:** Always prioritize readability
- **Don't Overuse Bright Colors:** Electric Cyan should be accent only
- **Don't Mix Print and Screen Values:** Use CMYK for print, RGB/HEX for screen
- **Don't Ignore Accessibility:** Always check color contrast ratios
- **Don't Use Pure Black:** Use Void Black (`#0A0A0F`) instead

---

### Typography Usage Rules

#### Do's ✅
- **Maintain Hierarchy:** Use consistent heading levels (H1→H2→H3)
- **Optimize Readability:** Follow line height guidelines (1.6 for body)
- **Load Fonts Efficiently:** Use `font-display: swap` for web performance
- **Test Across Platforms:** Verify rendering on different devices
- **Use Proper Weights:** Bold for headers, Regular for body
- **Limit Line Length:** Keep body text between 60-70 characters per line

#### Don'ts ❌
- **Don't Mix Too Many Fonts:** Stick to Space Grotesk + Inter + JetBrains Mono
- **Don't Use Below Minimum Sizes:** Maintain 14px minimum for body text
- **Don't Stretch Type:** Maintain proper aspect ratios always
- **Don't Ignore Line Length:** Long lines reduce readability
- **Don't Forget Fallbacks:** Include web-safe font fallbacks
- **Don't Use Decorative Fonts:** Keep it technical and professional

## Logo Recommendations

### Analysis of Current Logo (Kynet.svg)

**Strengths:**
- Modern, dynamic geometric design
- Strong purple gradient conveys premium, technical sophistication
- Scalable vector format
- Distinctive shape that stands out
- Good for abstract representation

**Areas for Improvement:**
- Missing wordmark integration ("AutoRail" text)
- No secondary variations (light/dark versions)
- Symbol alone may lack immediate brand recognition
- Gradient complexity may not reproduce well in all contexts
- No clear connection to "rail" or "infrastructure" metaphor

---

### Recommendations for Logo Enhancement

#### Priority 1: Develop Complete Logo System

**Create Wordmark Integration**
- Design "AutoRail" wordmark using Space Grotesk Bold
- Create multiple lockups: horizontal, vertical, stacked
- Ensure "Rail" portion can be emphasized with color/weight

**Logo Mark Variations**
- Simplify geometric shape for better recognition at small sizes
- Create solid-color versions (single-color Deep Rail Purple)
- Develop monochrome versions (black, white) for versatility
- Design favicon-optimized version (16×16px, 32×32px)

**Color Variations**
- Dark background version (with gradient)
- Light background version (solid Deep Rail Purple)
- Single-color version for print/embroidery
- Inverted version for special use cases

---

#### Priority 2: Strengthen Brand Metaphor

**Consider "Rail" Visual Elements**
- Incorporate subtle rail/track imagery into the geometric design
- Add horizontal lines suggesting infrastructure layers
- Include connection points representing system integration
- Maintain abstract aesthetic while hinting at core metaphor

**Infrastructure Symbolism**
- Add grid or circuit elements to represent backend systems
- Include flow indicators suggesting automation
- Develop animation potential (for web use) showing movement along "rails"

---

#### Priority 3: Improve Technical Clarity

**Simplification Options**
- Reduce gradient stops for easier reproduction
- Create clear vector outlines for better scalability
- Ensure 2-color version exists (purple + white/black only)
- Design outline-only version for special applications

#### Priority 4: Brand Consistency Across Platforms

- **Terminal:** ASCII-art logo for CLI branding

## Brand Voice & Messaging

### Brand Personality

**Attributes:**
- Technical yet approachable
- Confident but not arrogant
- Intelligent without being condescending
- Modern and forward-thinking
- Reliable and production-focused

**Tone Guidelines:**
- **Developer Documentation:** Clear, precise, technical
- **Marketing Copy:** Bold, confident, problem-focused
- **Product UI:** Concise, action-oriented, helpful
- **Error Messages:** Informative, solution-focused, supportive
- **Success Messages:** Brief, encouraging, professional

---

### Key Messaging

**Value Propositions:**
1. "Infrastructure on Autopilot" - Automated backend provisioning
2. "From Frontend to Full Stack" - Bridge the vibe-coding gap
3. "Production-Grade from Day One" - No technical debt accumulation
4. "Built for Agents" - Native support for agentic systems

**Taglines:**
- Primary: "Infrastructure on Autopilot for AI Agents"
- Secondary: "From Frontend to Full Stack in Seconds"
- Developer-focused: "Production-Grade Infrastructure, Automatically"
